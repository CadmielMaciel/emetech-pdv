Zona de perigo
Limpar banco de dados
Exclui permanentemente todas as empresas e todos os dados relacionados (produtos, clientes, vendas, OS, financeiro, usuários...). A empresa de sistema (admin_master) não é afetada e seu acesso ao painel é preservado. Esta ação não pode ser desfeita.
Asaas Master (cobrança das empresas)
Conecte a SUA conta Asaas master. É com ela que você gera as cobranças das empresas que gerencia (o dinheiro cai na sua conta). A chave fica protegida no servidor — depois de salva, não é exibida de volta.
No Asaas: Integrações → API → copie a chave. Deixe em branco para manter a atual.
Abrir Asaas
Para gerar PIX direto, cadastre uma chave PIX na sua conta Asaas master.
Administradores do Painel Master
Usuários com acesso irrestrito ao admin.html — gerenciam todas as empresas, planos e cobranças.
⚠️ Não confundir com os Admins de cada loja (PDV). Admin Master não acessa o PDV das lojas.
Adicionar Administrador Master
Cria a conta de login (e-mail + senha) e concede acesso master. O novo admin já pode entrar no painel com esses dados.

Migração X01 necessária
Rode este SQL no Supabase SQL Editor:
CREATE TABLE IF NOT EXISTS admin_master (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  nome text NOT NULL,
  email text UNIQUE NOT NULL,
  ativo boolean DEFAULT true,
  criado_em timestamptz DEFAULT now()
);
ALTER TABLE admin_master ENABLE ROW LEVEL SECURITY;
CREATE POLICY "master_all" ON admin_master FOR ALL USING (true) WITH CHECK (true);

Contrato do SaaS (você ↔ lojas)
Este é o seu contrato de prestação com cada loja-cliente — diferente dos modelos que as lojas usam com os clientes finais delas. Use variáveis que o sistema preenche por empresa:
{empresa} {documento} {plano} {valor} {periodicidade}
